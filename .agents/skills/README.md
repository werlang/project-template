# Project Skills

These skills describe the generic web/API template boundaries and workflow rules. Keep them aligned with the actual implementation when the template evolves.

- `test-first-delivery`: TDD Red-Green-Refactor workflow, quality contracts, testing decision tree, comment standards, Docker validation commands, and browser smoke checklists.
- `ai-workflow-update`: Agent context maintenance, auto-invocation rules, and ownership mapping for updating `AGENTS.md` and `.agents/skills/`.
- `documentation-maintenance`: Comprehensive consistency pass, owner mapping, mismatch ledgers, and terminology sweeps to keep docs, local skills, and code comments in sync.
- `document-touched-code`: JSDoc contract and intent comment requirements for touched source code.
- `bug-review`: Defect review stance for security, boundary enforcement, `data-*` attribute rules, SQL safety, and missing tests.
- `api-development`: Express route, middleware, and response conventions.
- `frontend-development`: Mustache, render middleware, Webpack, page script, helper, model, and component boundaries.
- `entity-models`: OOP entity model conventions and relation handling.
- `mysql-helper`: MySQL helper ownership and SQL boundary rules.
- `css-standards`: Shared tokens, page CSS, and component CSS rules.
